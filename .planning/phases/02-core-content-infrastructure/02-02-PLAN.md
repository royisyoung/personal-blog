In contentlayer.config.ts, define the `Post` document type with the following fields:

**Required fields (validation enforced):**

- `title`: string (required, short article title)
- `date`: date (required, publication date in ISO format YYYY-MM-DD)
- `description`: string (required, article description/excerpt for SEO and listing)
- `category`: string (required, primary category for the article)
- `tags`: list of strings (required, keywords/tags for filtering and related content)

**Computed fields:**

- `slug`: string (computed from the file directory name - each post is in `/content/posts/{slug}/index.mdx`, so slug = directory name)

**Configuration:**

- Set `directory: "posts"` under Post document type
- Include `include: ["**/*.mdx"]` - only process MDX files
- Use Zod validation provided by Contentlayer - if any field is missing or wrong type, build will fail
- Match decision: "Build fails if any post has invalid frontmatter" </action> <verify> <automated> grep -E "defineDocumentType|Post|title.*required|date.*required|slug" contentlayer.config.ts </automated> </verify> <done> - contentlayer.config.ts defines Post document type with defineDocumentType - title, date, description, category, tags are all required - tags field is defined as an array of strings - slug is a computed field from directory name - Post document configured to look in posts directory </done> </task>
