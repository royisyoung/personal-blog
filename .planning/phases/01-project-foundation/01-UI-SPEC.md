## Color

Following 60/30/10 rule (60% dominant, 30% secondary, 10% accent):

| Role | Value (Light) | Value (Dark) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | `#ffffff` | `#09090b` | Page background, main surfaces |
| Secondary (30%) | `#fafafa` | `#18181b` | Cards, navigation bars, sidebars |
| Accent (10%) | `#3b82f6` | `#60a5fa` | Links, primary buttons, interactive highlights |
| Destructive | `#ef4444` | `#f87171` | Destructive action buttons (reserved for future use) |

Accent reserved for:
- Inline text links within post content
- Primary call-to-action buttons
- Current page navigation indicator
- Focus rings on interactive elements

*Source: default neutral palette with blue accent for technical blog; dark mode configured via Tailwind class strategy per CONTEXT.md decision*

## Visual Hierarchy & Focal Point

> The default landing page (`app/page.tsx`) is created automatically by `create-next-app` scaffolding. This contract defines the minimal visual structure.

### Focal Point

**Primary focal point:** Site title/display heading at the top center of the viewport. This is the first thing users see when the page loads.

**Secondary focal point:** A short tagline/subheading below the title describing the site purpose.

**Visual priority flow:**
1.  Display heading (largest size, highest contrast)
2.  Tagline (medium size, lower contrast)
3.  Navigation links (if present in Phase 1)
4.  Footer copyright (smallest size, lowest contrast)

### Layout Structure

```
┌─────────────────────────────────┐
│  Navigation (if added later)    │
├─────────────────────────────────┤
│                                 │
│  [FOCAL] Title / Display        │
│  Tagline description            │
│                                 │
│  (empty for future content)     │
│                                 │
├─────────────────────────────────┤
│  Footer                         │
└─────────────────────────────────┘
```

- Centered content vertically and horizontally on initial load (landing page)
- Maximum content width constrained for readability
- Adequate whitespace around focal point to draw attention
